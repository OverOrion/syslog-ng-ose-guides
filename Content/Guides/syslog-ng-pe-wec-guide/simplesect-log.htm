Type: map

Description: The options to specify how to handle the internal logs of &wecabbrev;:

&wecabbrev; sends internal log messages to stderr. You can also optionally specify a to send logs to (in parallel with stderr). If you are using a systemd platform and start &wecabbrev; using systemctl, then stderr is redirected to systemd-journal, and this is where you will find the internal logs of &wecabbrev;.

level
Type: debug|info
Default: info

Description: The application log level of &wecabbrev;.

Possible values are:

  • debug: Information with the most details, useful when debugging &wecabbrev; and diagnosing issues.

  • info: Basic information about the initialization of &wecabbrev;. Following initialization, no information is displayed on screen, unless an issue occurs.
file
Type: string
Default: N/A

Description: The path to the file where &wecabbrev; should write internal log messages. The log file is automatically created by &abbrev;.

You can send this file to syslog-ng using a file() source.
