Licensing model and modes of operation

A Log Source Host (LSH) is any host, server, or device (including virtual machines, active or passive networking devices, syslog-ng clients and relays, and so on) that is capable of sending log messages. Log Source Hosts are identified by their IP addresses, so virtual machines and vhosts are separately counted.

The application has three distinct modes of operation: Client, Relay, and Server.

  • In Client mode collects local logs generated by the host it is running on, and forwards them through a network connection to the central server, a relay, or another network destination. If you install the application in Client mode on a host, it counts as a Log Source Host, even if it does not send log messages to a server.

  • In Relay mode receives logs through the network from Log Source Hosts and forwards them to the central server, a relay, or another network destination. If you install the application in Relay mode on a host, it counts as a Log Source Host, even if it does not send log messages to a server.

    Relays cannot store the received log messages in local files, except for the log messages of the relay host. Naturally, relays can use the disk-buffer option for every message.

  • In Server mode acts as a central log-collecting server that receives messages through a network connection, and stores them locally, or forwards them to other destinations or external systems (for example, a SIEM or a database). Installing the application in Server mode requires a license file, this license file determines the number of Log Source Hosts that can send log messages to the server.

Notes about counting the licensed hosts

Note that the number of source hosts is important, not the number of hosts that directly sends messages to : every host that send messages to the server (directly or using a relay) counts as a Log Source Host.

  • If the actual IP address of the host differs from the IP address received by looking up its IP address from its hostname in the DNS, the syslog-ng server counts them as two different hosts.

  • If the no-parse flag is set in a message source on the server, assumes that the message arrived from the host (that is, from the last hop) that sent the message to , and information about the original sender is lost.
