snmptrap() source options

The snmptrap() driver has the following options. Only the filename() option is required, the others are optional.

filename()
Type: path
Default:

Description: The log file of snmptrapd. The application reads the traps from this file.

Default value: .snmp. option.

set-message-macro()
Type: yes|no
Default: yes

Description: The snmptrap() source automatically parses the traps into name-value pairs, so you can handle the content of the trap as a structured message. Consequently, you might not even need the ${MESSAGE} part of the log message. If set-message-macro() is set to no, leaves the ${MESSAGE} part empty. If set-message-macro() is set to yes, generates a regular log message from the trap.
