syslog-ng(): Forward logs to another syslog-ng node

The syslog-ng() destination driver forwards log messages in a special format to another syslog-ng node. This format (called Enterprise-wide message model or EWMM) can transfer log messages between syslog-ng nodes so that at the last node, the original message can be reconstructed. It also allows you to transfer the name-value pairs extracted on the first syslog-ng node that received the message to your syslog-ng server.

The syslog-ng() destination driver is available in version 7.0.93.16 and later. The node that receives this message must use the default-network-drivers() source to receive the messages.
