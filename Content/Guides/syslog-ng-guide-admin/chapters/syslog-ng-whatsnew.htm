What is new in version ?

This section lists the most recent changes of ().

  • New source: mqtt()

    You can use the mqtt() source to fetch messages from MQTT brokers.

  • New destination: discord()

    The discord() destination driver sends messages to Discord using Discord Webhook.

  • New parser: regexp-parser()

    The application can parse fields from a message with the help of regular expressions. This can be also achieved with the match() filter, by setting the store-matches flag, but the regexp-parser() offers more flexibility, like multiple patterns and setting the prefix of the created name-value pairs.

  • New filter: rate-limit()

    Limits messages rate based on arbitrary keys in each message.

  • New options for the kafka() destination C implementation

    Options batch-lines() and batch-timeout() have been added.

  • Proxy Protocol support

    Proxy Protocol is now supported in the network() source.

    If you connect load balancers to your application, identifies every connection that is connected to the load balancers identically by default, regardless of the source IP or the source protocol. Essentially, the load balancer masks the source IP unless you enable Proxy Protocol support for your proxy TLS transport() to inject information about the original connection into the forwarded TCP session.

  • New --remove-orphans option in syslog-ng-ctl stats

    New option --remove-orphans has been added to the stats command.

  • New options for the mongodb() destination

    Options collection() and workers() have been added.

  • New monitoring() source driver

    The monitoring() source allows you to select which statistics of you want to monitor. In addition, the statistics are available as structured name-value pairs, so you can format the output similarly to other log messages. That way, you can easily convert the statistics and metrics, for example, into JSON or WELF format. That way, you can send the statistics of your log messages into a monitoring solution, for example, into Riemann, Redis, or Graphite.
