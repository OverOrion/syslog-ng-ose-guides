Possible error messages you may encounter while using the mqtt() destination

status_code Complete error message Possible reason(s) Possible solution(s)
Van esetleg ilyenünk? 
"ERROR, while init threaded dest. ..."

The application will not start.

Itt van ötletünk, mit tud csinálni (ha tud bármit)?
Van esetleg ilyenünk? 
"mqtt: thje topic() argument is required for mqtt destinations. ..."

The topic option is not set in your configuration. The application will not start.

Itt állítsa be a topic()-ot a configban, és indítsa újra a syslog-ng-t?

Van esetleg ilyenünk? 
"The mqtt destination does not support the batching of messages, ..."

batch-timout and/or batch-lines can not be set. The application will not start.

Itt arról van szó, hogy ELEVE nem támogatja a batch-timeout() / batch-lines() funkciókat (ahogy az error msg sugallja jelenlegi formájában), vagy MOST nem tudja valamiért kezelni?

Ha esetleg a fenti opciók valamelyikénél csűrt el valamit pl. configban, akkor azzal tud-e valamit csinálni? (És honnan fogja tudni, hogy MIT csináljon?)

Van esetleg ilyenünk? 
"Disconnected during publish!"

The application can not send the message, because (?) disconnected, but it (?) will try to resend the message.

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Kérdés: MI disconnected? A syslog-ng fog próbálkozni az üzenetek újraküldésével, ugye?
Van esetleg ilyenünk? 
"Max message inflight! (publish)"

The application can not send the message, broker response code is: max message inflight, it (?) will try to resend the message.

KÉRDÉS: itt van egy külön broker response code, amit esetleg említeni kéne valahogy?

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Van esetleg ilyenünk?

 

"Failure during publishing!"

The application can not send the message, broker response code is: failure, it (?) will try to resend the message.

KÉRDÉS: itt van egy külön broker response code, amit esetleg említeni kéne valahogy?

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Van esetleg ilyenünk?

 

"Error during publish!"

The application can not send the message AND the message is dropped. Possible reason: bad_utf8_string (topic), NULL parameter.

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Pl. át kéne írnia a topic()-ban a nevet? 

Van esetleg ilyenünk?

 

"Disconnected while waiting the response!"

The application (?) can not get any response, because client is disconnected from broker. It (?) will resend the message.

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Van esetleg ilyenünk?

 

"Error while wating the response!"

The application (?) can not get any response, broker response code: failure. It (?) will resend the message.

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Van esetleg ilyenünk?

 

"Error constructing topic ..."

wrong topic template, it (?) will use the fallback option.

MI használja a fallback-topic() (?) opciót, a syslog-ng / a destination?

Itt van ötletünk, mit tud csinálni (ha tud bármit)?

Ha nincs esetleg beállítva fallback-topic() (bár ha jól sejtem, ugyanúgy kötelező lenne beállítania itt is, ha a topic() esetében nem stringet használ, hanem macro-t, vagy template function-t, mint ahogy a kafka-c() esetében?), akkor mi történik? 

Van esetleg ilyenünk?

 

"mqtt dest: topic name is illegal, it can't be empty"

the topic template option gives back 0 length string, will use the fallback topic.

Kb. mint a fenti kérdés

Van esetleg ilyenünk?

 

"Error connecting mqtt client ..."

can not connect to broker, it (?) will try later.

Tud bármit tenni, vagy várnia kell, aztán talán megjavul a broker?

Van esetleg ilyenünk?

 

"Error creat mqtt client ..."

error while creating the client, it (?) will try later. wrong address, broker isn't running.

Le tudja esetleg valahogy ellenőrizni, hogy MELYIK igaz? Gondolom, az address-t még könnyebb leellenőrizni, de honnan tudhatja esetleg, ha a broker fosta össze magát?
