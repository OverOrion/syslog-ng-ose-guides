Resolved Issue Issue ID

Support for cap_syslog capability

The capability library (libcap) shipped with syslog-ng PE was outdated and it did not support the cap_syslog capability. From now on, it is supported.

 SYSLOGDEV-4381

JVM not unloaded at syslog-ng reload

Using Java-based destinations, when syslog-ng was reloaded, the Java Virtual Machine (JVM) was not exited so extra java processes remained. From now on, during a reload, the old java process will remain and it will be used after the reload.

 SYSLOGDEV-4368

Logstore file owner not set correctly

When syslog-ng was running with only user privileges but with CAP_CHOWN capability enabled, the owner of the logstore file was not set correctly. This issue has been fixed.

SYSLOGDEV-4361

syslog-ng crash when trying to open a corrupt logstore

In certain cases, when an attempt was made to open a corrupt logstore, syslog-ng crashed. In such cases, a core dump is now generated.

SYSLOGDEV-4350

Minor fixes in SELinux installer script

This fix solves a few minor bugs in the SELinux installer script.

SYSLOGDEV-4345

Messages from journald are not included in center statistics

The center counter of the statistics framework did not count the messages arriving from the journal source. This issue has been fixed.
