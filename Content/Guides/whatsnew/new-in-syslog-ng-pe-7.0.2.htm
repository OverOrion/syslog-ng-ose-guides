Highlights of 7.0.2

Python support: message parsers and template functions

The Python Log Parser allows you to write your own parser in Python. Practically, that way you can process the log message (or parts of the log message) any way you need. For example, you can import external Python modules to process the messages, query databases to enrich the messages with additional data, and many other things. For details, see .

You can write your own template function in Python. You can define a Python block in your configuration file, and define one or more Python functions in it. You can use these functions as template functions. For details, see the manual pages.

Monitor more effectively

The new monitoring() source allows you to granularly select which statistics of you want to monitor. In addition, the statistics are available as structured name-value pairs, so you can format the output similarly to other log messages. That way, you can easily convert the statistics and metrics, for example, into JSON or WELF format, and send the results into your monitoring database. For details, see .

version 7.0.2 also includes the monitoring-welf() source, which is a preconfigured monitoring() source that generates statistics messages in WELF format. Starting with version 7.0., uses this driver for new installations to generate statistics (earlier versions use the internal() source for this purpose).

Ported from the 6 LTS product line

  • The functionality of syslog-ng-query application is available in 7.0.2, as part of the syslog-ng-ctl utility.
