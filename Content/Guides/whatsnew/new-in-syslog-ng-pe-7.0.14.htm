New features in 7.0.14

Google Stackdriver destination

The stackdriver destination of can send log messages to the Google Stackdriver cloud. Google Stackdriver is a widely used metrics, event, and log aggregator and analyzer system. For details, see .

Elasticsearch HTTP destination

Version 7.0.143.21 of can directly post log messages to an Elasticsearch deployment using the Elasticsearch Bulk API over the HTTP and Secure HTTP (HTTPS) protocols. For details, see .

Windows Server 2019 logs

The application now supports Windows Server 2019.

Enhancements

  • The syslog-ng-ctl reload command now has a return value: 0 if the operation was successful, 1 otherwise.

  • You can use the guess-timezone source flag to guess the timezone of the message if this information is not available in the message.

  • Instead of dropping incoming messages that are too long, you can now trim them using the trim-large-messages() option of the syslog() source.
