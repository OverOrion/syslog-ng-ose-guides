Globalization

Technical writers should use the text automatically generated from the Avenue Product Release Calendar.

The Product Release Calendar uses an icon to indicate whether globalization information is available. If the circle on the icon is green, the globalization information has been entered and approved and can be used in a statement.

If your product does not have a green Internationalization icon, you will need to talk to the development manager so that they can update the information and get it approved.

To obtain the Global Operation statement

  1. Go to the Avenue main page, and click Products | Product Release Calendar | Release Listing.
  2. Locate your product.
  3. Click the Release Tasks icon, then click Globalization.
  4. In the Globalization Checklist page, select View Global Operation Statement.
  5. Copy the Global Operation Statement text to Notepad, and then to the Global Operations section of your Release Notes.

This section contains information about installing and operating this product in non-English configurations, such as those needed by customers outside of North America. This section does not replace the materials about supported platforms and configurations found elsewhere in the product documentation.

This release is Unicode-enabled and supports any character set. In this release, all product components should be configured to use the same or compatible character encodings and should be installed to use the same locale and regional options. This release is targeted to support operations in the following regions: North America, Western Europe and Latin America, Central and Eastern Europe, Far-East Asia, Japan. It supports bidirectional writing (Arabic and Hebrew). The release supports Complex Script (Central Asia – India, Thailand).
