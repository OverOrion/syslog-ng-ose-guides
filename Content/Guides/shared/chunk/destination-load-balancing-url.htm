Starting with version 3.197.0.12, you can specify multiple URLs (separated with a whitespace). In this case, sends log messages to the specified URLs in a load-balance fashion. This means that sends each message to only one URL. For example, you can use this to send the messages to a set of ingestion nodes or indexers of your SIEM solution if a single node cannot handle the load. Note that the order of the messages as they arrive on the servers can differ from the order has received them, so use load-balancing only if your server can use the timestamp from the messages. If the server uses the timestamp when it receives the messages, the order of the messages will be incorrect.
