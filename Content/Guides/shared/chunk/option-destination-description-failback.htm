When starts up, it always connects to the primary server first. The failback() option determines what happens after fails over to a secondary server.

  • If failback() is set, attempts to return to the primary server.

    After connects a secondary server during a failover, it sends a probe (a TCP SYN packet) every tcp-probe-interval() seconds (if the network timeout value in the kernel is set to a longer value, sends the packet using the kernel settings). If the primary logserver responds with a TCP ACK packet, the probe is successful. When the number of successful probes reaches the value set in the successful-probes-required() option, reconnects the primary server.

  • If failback() is not set, does not attempt to return to the primary server even if it becomes available. In case the failover server fails, attempts to connect the next failover server in the list in round-robin fashion. This is the default behavior in version 7.0.9 and earlier.
