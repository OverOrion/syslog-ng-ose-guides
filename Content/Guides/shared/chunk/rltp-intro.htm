The application can send and receive log messages in a reliable way over the TCP transport layer using the (). is a proprietary transport protocol that prevents message loss during connection breaks. The transport is used between hosts (for example, a client and a server, or a client-relay-server), and interoperates with the flow-control and reliable disk-buffer mechanisms of , thus providing the best way to prevent message loss.

The sender detects which messages the receiver has successfully received. If messages are lost during the transfer, the sender resends the missing messages, starting from the last successfully received message. Therefore, messages are not duplicated at the receiving end in case of a connection break (however, in failover mode, this is not completely ensured). also allows for encrypted and non-encrypted connections to be received on the same port, using a single source driver.
