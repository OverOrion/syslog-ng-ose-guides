failover-servers() DEPRECATED
Type: list of IP addresses and fully-qualified domain names
Default: 0

Description:This option is deprecated and will be removed from future releases. Use the failover() option instead.

Available only in version 3.2 and later. Specifies a secondary destination server where log messages are sent if the primary server becomes inaccessible. To list several failover servers, separate the address of the servers with comma. The time waits for the a server before switching to the next failover server is set in the time-reopen() option. For details about how client-side failover works, see Client-side failover.

 Caution:

The failover servers must be accessible on the same port as the primary server.

NOTE:

This option is not available for the connection-less UDP protocol, because in this case the client does not detect that the destination becomes inaccessible.
