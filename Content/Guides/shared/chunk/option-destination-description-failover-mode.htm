When starts up, it always connects to the primary server first. The failover-mode() option determines what happens after fails over to a secondary server.

  • If failover-mode() is set to failback, attempts to return to the primary server every time-reopen() seconds (by default, every ten minutes). This is the default behavior in version 7.0.10 and later.

  • If failover-mode() is set to round-robin, does not attempt to return to the primary server even if it becomes available. This is the default behavior in version 7.0.9 and earlier.
